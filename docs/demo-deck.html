02 — UX Decisions
StreamCal
Design Philosophy

Every decision reduces cognitive load.

Decision 01
📅 Calendar as Home
A temporal map instead of a list. You see what's happening today, this week, coming soon — no mental math required. Answers the user's real question: "what do I watch tonight?"
Decision 02
➡️ Forward-Looking Only
We removed "mark as watched." StreamCal isn't a logging app — it's an anticipation engine. Simpler data model, cleaner screens, and it keeps users focused on what's next.
Decision 03
✦ AI With Real Context
Ask StreamCal sends your actual library as context to Claude. It can say "Severance S3E5 airs Thursday" — not generic recs. It also recommends beyond your library, informed by your taste.
Decision 04
🎨 Semantic Design System
Shows = purple, Movies = red, Sports = green, AI = indigo — consistently across every screen. One DS enum controls all spacing, radii, and color. Zero magic numbers.
03 — Technical Stack
StreamCal
Architecture

Native. No dependencies.
AI-first from day one.

Layer Technology Why It Matters
UI + Data SwiftUI + SwiftData Declarative, no ORM boilerplate, cascade deletes built-in
TV + Movies TMDB API 1M+ titles, real air dates, streaming providers, poster art
Anime AniList GraphQL Purpose-built anime DB with per-episode schedules
Sports ESPN + TheSportsDB Live game schedules across every major league
AI Claude Haiku via Cloudflare API key never in binary; Pro entitlement verified server-side
Monetization RevenueCat Cross-platform subscription + receipt validation

🔐 Security First

Claude API key lives in Cloudflare Workers — never ships in the app binary. Worker validates RevenueCat Pro entitlement before forwarding any request.

⚡ Zero Third-Party UI Deps

Custom CachedAsyncImage with NSCache, custom FlowLayout, custom design system. No SPM bloat.

🔄 Stateless Refresh

RefreshService re-fetches all APIs on launch and reschedules notifications atomically. No stale cache, no manual sync button.
