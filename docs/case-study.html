Evolution of Project

Where it started

The original concept was narrow: a simple episode tracker for TV shows. Search for a show, see upcoming episodes, set a reminder.

  • Target user: someone actively watching 3–5 shows at once
  • Core loop: Search → Save → See air date
  • Single data source: TMDB API for episodes
  • No AI, no sports, no movies
  • Inspired by the friction of keeping up with shows across platforms
V1 Scope
📺 TV Shows only

Episode list, air dates, push reminders. One source: TMDB.

🎬 Movies — not yet

No theatrical or streaming release date tracking.

🏆 Sports — not yet

No team schedules, no ESPN integration.

✨ AI — not yet

No Claude integration, no discovery, no personalization.

Evolution of Project

How the scope evolved

1

TV Shows only

Basic episode tracker. TMDB search, upcoming episodes list, push reminders at air time.

2

+ Movies & Anime

Realized movies and anime have the same core need: release dates, reminders, a library to track progress. Added TMDB movie support and AniList for anime.

3

+ Sports & Calendar-first home

Sports teams added via ESPN + TheSportsDB. Calendar became the home screen — the answer to "what do I need to care about today" on open.

4

+ AI Discovery & Freemium

Claude AI assistant added for scheduling and discovery. RevenueCat paywall gates AI behind Pro. Trakt OAuth integration for watch history sync.

Technology Decisions

SwiftUI + SwiftData

Choosing native frameworks wasn't the default — it was a philosophy. No third-party UI library means the app looks and feels like iOS, not like a cross-platform wrapper.

WHY IT WORKED
  • Declarative UI = rapid iteration
  • @Query auto-updates UI when data changes
  • Zero ORM setup — models are just annotated classes
  • SwiftUI animations are first-class
  • No compatibility shims for Apple APIs
CHALLENGES
  • SwiftData complex query limitations
  • Migration edge cases required careful defaults
  • Some SwiftUI layout bugs needed workarounds

Technology Decisions

API decisions

API Purpose Why Selected Challenges Score
TMDB Shows, movies, episodes, watch providers, posters Industry standard, free tier generous, excellent watch-provider endpoint None significant 9
ESPN API North American pro sports schedules, scores, broadcast networks Only free source for NFL/NBA/MLB/NHL with broadcast data Undocumented — response shapes vary by endpoint, required defensive decoding 7
AniList GraphQL Anime metadata, episode air dates, next airing Clean schema, comprehensive, no API key required GraphQL pagination edge cases with large catalogs 8
TheSportsDB International soccer + fallback sports data Best coverage for Premier League, La Liga, Champions League Free tier rate-limited to ~1 req/350ms — required careful throttling 6

Key lesson: ESPN's undocumented API required building a defensive JSON decoder that handles inconsistent response shapes across endpoints — a real-world engineering skill that wouldn't come from a documented API.

Technology Decisions

AI + Monetization

CLAUDE AI — 9/10

Returns structured JSON, not prose. The app renders native cards — not a chat window — for a significantly better user experience.

  • API key held server-side in Cloudflare Worker — never touches the client
  • Worker validates RevenueCat Pro entitlement before forwarding request
  • Two modes: library-aware scheduling vs. open-ended discovery
  • Structured JSON schema enforced in system prompt
REVENUECAT — 10/10

Eliminated the hardest part of App Store monetization — receipt validation, entitlement management, and the customer subscription portal.

  • Single SDK call to check isPro entitlement anywhere in the app
  • Built-in customer center for managing / canceling subscriptions
  • Would use again on every paid iOS app

Why the Worker pattern?
API key never shipped in the app binary. Zero cost to rotate it. Pro check runs server-side — can't be bypassed by the client.

Reflections

What we learned.
What's next.

WHAT WENT WELL
  • Native-first approach delivered a genuinely polished, iOS-quality feel
  • Structured AI responses were the right call — cards beat chat bubbles every time
  • RevenueCat removed months of subscription management complexity
  • Cloudflare Worker pattern kept API keys off the client cleanly
  • Calendar-as-home answered the core question immediately on open
WHAT WE'D DO DIFFERENTLY
  • Start with a design token system earlier — retrofitting consistency took effort
  • Evaluate ESPN alternatives sooner — undocumented APIs slow iteration
  • Scope AI features first, not last — they shaped the product more than expected
WHAT'S NEXT
  • Home screen widget — today's schedule at a glance
  • AI-generated weekly push notification recap
